Chapter 2 Reading Notes

1 Chapter 2 Intelligent Agents

1.1 Agents

1.1.1 Agent:

  1. sensors: perceiving environment
    • percept: agent's perceptual input at any given instant
    • percept sequence: complete history of everything perceived
  2. actuators: acting upon environment
    • agent function: mapping any percept sequence to an action (abstract)
    • agent program: implementation of agent function (concrete)

1.2 Rationality

1.2.1 Rational Agent

  1. Performance Measure:
    • desirability, evaluating any given sequece of environment states
    • if it evaluates agent states, agent can "fool" itself
  2. Definition of Rational Agent:

    For any possible percept sequence, a rational agent should select an action that is expected to maximizie its performance measure.

1.2.2 Rationality vs Omniscience

  • Rationality maximizes expected performance
  • Omniscience: knowing actual outcome –> perfection, maximizing actual performance
  • Information Gathering: doing action in order to modify future percepts

1.2.3 Autonomy

  • autonomous agent: should learn what it can to compensate for partial/incorrect prior knowledge

1.3 Environments

1.3.1 Specifying the task environment

  • PEAS description: Performance, Environment, Actuators, Sensors
  • Properties of Task Environments:
    • Observable: fully vs partially vs unobservable
    • Agent Number: single vs multi

      For multi-agent environment:

      • competitive (Eg: Chess)
      • partially cooperative (taxi-driving)
    • Deterministic: deterministic vs stochastic vs uncertain
      • deterministic: next state completely determined by current state and action
      • uncertain: not fully observable or not deterministic
      • non-deterministic: no probabilites attached to action
    • Future: episodic vs sequential
      • episodic: next episode does not depend on actions in previous episodes

        Eg: many classification tasks

      • sequential: current decision could affect all future decisions
    • Change: static vs dynamic vs semidynamic
      • dynamic: environment can change while an agent is deliberating
      • semidynamic: agent's performance changes with the passage of time

        Eg: chess played with a clock

    • Time: discrete vs continuous
      • discrete: finite number of distinct states. Eg: chess without timer
    • Knowledge: known vs unknown
      • known: outcomes of actions are given, rules are given
      • unknown: agent has to learn in order to make good decisions

1.4 Structure of Agents

1.4.1 Overview

  • AI: design an agent program on some architecture to implement agent function
  • architecture: computing device with physical sensors and actuators
  • \(agent = architecture + program\)

1.4.2 Different types of Agents

  1. Table Driven Agengts
    • \(action \leftarrow LOOKUP(percepts, table)\)
    • look up table contains too many entries, impossible to implement
  2. Simple Reflex Agents
    • selecting actions based on current percept, ignoring rest of history
    • condition-action rule
      • = situation-action rules = productions = if-then rules
      • build a general purpose interpreter for condition-action rules
      • then create rule sets for specific task environments

        \(state \leftarrow INTERPRET-INPUT(percept)\)

        \(rule \leftarrow RULE-MATCH(state, rules)\)

        \(action \leftarrow rule.ACTION\)

    • Problems

    Works well only with fully-observable environment

    • Vacuum Cleaner Example: without [Location] sensor, infinite loop can occur
    • \(Randomization\) can help excape infinite loops
  3. Model-based Reflex Agents
    • Model: knowledge about "how the world works"
    • Internal State: depends on percept history, agent uses it to keep track of the world

      \(state \leftarrow UPDATE-STATE(state, action, percept, model)\)

      \(rule \leftarrow RULE-MATCH(state, rules)\)

      \(action \leftarrow rule.ACTION\)

  4. Goal-based Agents
    • goal: describing desirable situations
    • serach(C3~C5) and Planning(C10, C11)
  5. Utility-based Agents
    • How to distinguish between different action sequences leading to the same goal?

      –> "How happy will I be in such state?"

      –> Utility

    • Utility Function: internalization of performance measure
    • Expected Utility:
      • Due to partial observability and stochasticity in the real world
  6. Learning Agents: all agents above can improve performance through learning
    • Performance Element

      selecting external actions based on instalation of rules

    • Learning Element

      making improvments, formulating rules

    • Critic

      tells the learning element how well agent is doing, with respect to fixed performance standard

    • Problem Generator

      suggesting actions that will lead to new and informative experiences

1.4.3 How the components of agent programs work

  • Categorize them in increasing expressiveness
  1. Atomic
    • each state of the world is indivisible, has no internal structure
    • like a "black box" with only one property: identical to vs different from another atom
  2. Factored
    • splits each state into a fixed set of variables/attributes with values
    • uncertainty: leaving an attribute blank
  3. Structured
    • each state has objects, each can have their own attributes and relationships to other objects
    • almost everything humans express in natural language
